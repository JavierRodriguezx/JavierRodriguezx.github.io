¿Qué es JavaScript?

En el núcleo de la programación web moderna, encontramos a JavaScript: un lenguaje de programación de alto nivel, interpretado y orientado a objetos. Su versatilidad y capacidad para ejecutarse en el navegador del usuario lo convierten en un elemento indispensable para la creación de experiencias interactivas y dinámicas en la web.


  • Diseño y Desarrollo Web.
  • Hacer Videojuegos.
  • Experiencias 3D, Realidad Aumentada, Realidad Virtual.
  • Controlar Hardware (drones, robots, electrodomésticos, wearables, etc).
  • Aplicaciones Híbridas y Móviles.
  • Aprendizaje Automático.
  • etc.

Caracteristicas

  • Lenguaje de Alto Nivel.
  • Interpretado.
  • Dinámico.
  • Débilmente Tipado.
  • Multi paradigma.
  • Sensible a MAYÚSCULAS y minúsculas.
  • No necesitas los puntos y comas al final de cada línea.

Historia y Evolución de JavaScript

Nos transportamos a la década de 1990, una época en la que la web estaba dominada por páginas estáticas. En 1995, el genio programador Brendan Eich introdujo JavaScript con el objetivo de dotar a las páginas web de interactividad. Desde entonces, JavaScript ha experimentado un desarrollo continuo, incorporando nuevas características y optimizaciones para adaptarse a las crecientes demandas del desarrollo web contemporáneo.


¿Cómo se ejecuta JavaScript en un navegador?

Ahora, adentrémonos en la arquitectura subyacente que permite la ejecución de JavaScript en los navegadores modernos. Al abrir una página web, el navegador descarga el código fuente HTML y, al encontrar referencias a scripts JavaScript, desencadena el proceso de ejecución. Aquí es donde entra en juego el motor JavaScript del navegador, encargado de interpretar y ejecutar el código. Este proceso implica la creación de objetos en el DOM, permitiendo la manipulación dinámica de la interfaz de usuario y la interacción del usuario con la página web.

Explorar JavaScript no es solo descubrir un lenguaje de programación; es desentrañar el tejido mismo de la web moderna. Desde la manipulación del DOM hasta la ejecución eficiente en el navegador, JavaScript es la fuerza impulsora detrás de la interactividad que damos por sentada en cada clic y desplazamiento.

Variables

Comencemos explorando el concepto de variables. Imagina que son como cajas virtuales donde almacenamos información esencial para nuestros programas en JavaScript. Estas cajas, creadas mediante las palabras clave var, let o const, actúan como contenedores con nombres específicos.

let, const y var son palabras clave utilizadas para declarar variables en JavaScript, pero tienen comportamientos y ámbitos diferentes. Aquí hay una descripción de cada una:


Var (Global):

  • Forma antigua de declarar variables en JavaScript.
  • Tiene ámbito de función y puede ser global si se declara fuera de funciones.
  • No tiene ámbito de bloque.
  • Se "eleva" durante el hoisting.

Let:

  • Introducido en ES6 para mejorar el manejo del ámbito.
  • Tiene ámbito de bloque.
  • No se "eleva" de la misma manera que var.

Const:

  • Introducido en ES6 para declarar variables con valores inmutables.
  • Tiene ámbito de bloque.
  • No se "eleva" de la misma manera que var.

Comprender los diferentes tipos de operadores es esencial para escribir código eficiente y funcional.

Estructura de control (if, else, else if, switch, for, while y do while).

En programación, las estructuras de control son herramientas fundamentales que permiten dirigir el flujo de ejecución de un programa. Estas estructuras ayudan a tomar decisiones y repetir acciones, lo que proporciona flexibilidad y control sobre el comportamiento del código. Las estructuras de control se dividen comúnmente en dos tipos: condicionales y bucles.

Tomando Decisiones con "if, "else" y "else if".

En el vasto universo de las estructuras condicionales, nos encontramos con el if, el guardián de las decisiones. Cuando queremos que una parte específica del código se ejecute solo si se cumple una condición, recurrimos a este fiel compañero. Pero, ¿qué pasa si la condición no se cumple? Aquí es donde entra en juego el else, listo para manejar el caso contrario. Pero, ¡ah! ¿Y si hay más de una condición que queremos evaluar? No te preocupes, el else if está ahí para resolver ese dilema. Vamos a verlo en acción con un ejemplo práctico:

JavaScript

let edad = 18;

if (edad >= 18) { // Permite ejecutar un bloque de código si una condición es verdadera.

console.log("Eres mayor de edad. ¡Bienvenido!");

} else {

console.log("Eres menor de edad. Debes esperar un poco más.");

Tomando Diferentes Caminos con 'switch', 'break' y 'continue'

En ocasiones, queremos tomar diferentes caminos según el valor de una variable. Aquí es donde entra en juego el lector de opciones, el switch. Pero, ¿y si queremos salir de un bucle antes de que termine o saltar a la siguiente iteración? Ahí es donde entran en juego break y continue. Veamos un ejemplo que utiliza estas estructuras para gestionar diferentes casos:

Repitiendo Tareas con 'for', 'while' y 'do-while'

Los bucles son como el pulso de nuestro programa, permitiéndole repetir tareas una y otra vez. Comencemos con el confiable for, ideal para iterar sobre secuencias de datos. Luego, el leal while se presenta, ejecutando un bloque de código mientras una condición sea verdadera. Y, para asegurarnos de que el código se ejecute al menos una vez, ahí está el valiente do-while. Vamos a darles vida con un ejemplo que imprime los primeros números naturales:

'for'

El bucle for en JavaScript es conocido por su capacidad de proporcionar un control preciso sobre la iteración. Consideremos este ejemplo, donde iteramos sobre los primeros cinco números naturales:

JavaScript

for (let i = 1; i <= 5; i++) {

console.log(i);

}

En este caso, let i = 1 inicializa la variable de control i, la condición i <= 5 determina cuántas veces se ejecutará el bucle, y i++ incrementa i en cada iteración

'While'

El bucle while es ideal cuando la repetición depende de una condición booleana. Aquí, imprimimos los números pares hasta 10:

JavaScript

let numero = 2;

while (numero <= 10) {

console.log(numero);

numero += 2;

}

Este bucle continúa ejecutándose mientras la condición numero <= 10 sea verdadera, proporcionando una flexibilidad dinámica.

'Do-while'

El bucle do-while asegura que su bloque de código se ejecute al menos una vez, incluso si la condición es falsa más adelante. En este ejemplo, solicitamos al usuario un número mayor que 100:

JavaScript

let entrada;

do {

entrada = prompt("Ingrese un número mayor que 100:", "");

} while (entrada <= 100 && entrada);

La condición entrada <= 100 && entrada se evalúa después de la primera ejecución, asegurando la ejecución mínima.

'for-of': Explorando Elementos de una Colección

El bucle for-of es especialmente útil para recorrer elementos de colecciones, como arrays. Aquí, imprimimos los colores de un array:

JavaScript

let colores = ["rojo", "verde", "azul"];

for (let color of colores) {

console.log(color);

}

Este bucle simplifica el acceso y procesamiento de elementos en estructuras de datos iterables.

Bucle 'for-in': Iterando sobre Propiedades de Objeto

El bucle for-in en JavaScript se utiliza para iterar sobre las propiedades enumerables de un objeto. A diferencia de for-of, que se utiliza para arrays y otras estructuras iterables, for-in es especialmente útil cuando trabajamos con objetos y queremos acceder a sus propiedades. Aquí hay un ejemplo:

JavaScript

let colores = ["rojo", "verde", "azul"];

for (let color in colores)

console.log(color);

}

En este caso, el bucle for-in itera sobre las propiedades de persona, y para cada iteración, propiedad toma el nombre de la propiedad actual. La expresión persona[propiedad] accede al valor asociado a esa propiedad.

Este bucle es particularmente útil cuando necesitamos examinar dinámicamente las propiedades de un objeto y realizar acciones en consecuencia.

Descubre cómo estas herramientas impulsan la modularidad en el desarrollo.

Declaracion

La declaración de función crea una función con un nombre específico que puede ser llamada en cualquier parte del código, incluso antes de su declaración debido al concepto de elevación (hoisting).

JavaScript

function saludar(nombre) {

console.log("Hola, " + nombre + "!");

}

Expresion de funcion.

Las expresiones de función se invocan utilizando la variable a la que se asignan.

JavaScript

let sumar = function (a, b) {

return a + b;

}

// Invocación

let resultado = sumar(3, 5);

console.log(resultado); // Resultado: 8

Funciones Que retornan un valor

Estas funciones se invocan de la misma manera que otras funciones, y el resultado se puede almacenar en una variable para ser mostrado el return.

JavaScript

function multiplicar(a, b) {

return a * b;

// Invocación

let resultado = multiplicar(4, 6);

console.log(resultado); // Resultado: 24

Funciones Anonimas

Las funciones anónimas se invocan usando la variable a la que están asignadas, seguida de paréntesis y argumentos.

JavaScript

let saludar = function(nombre) {

console.log("Hola, " + nombre + "!");

};

// Invocación

saludar("Ana");

Callbacks:

Los callbacks se pasan como argumentos y se invocan después de que ocurre un evento o se completa una tarea.

JavaScript

function operacionAsincrona(datos, callback) {

// Realiza una operación asíncrona y luego llama al callback

setTimeout(function() {

callback(datos);

}, 1000);

}

// Invocación con Callback

operacionAsincrona({ mensaje: "Hola, mundo!" }, function(datos) {

console.log("Datos recibidos:", datos);

});
